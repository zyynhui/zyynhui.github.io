pack

  1. 使用webpack 配置步骤
    ① npm init -y ②建文件 dist（上线文件） src 源代码（ index.html main.js ）
    根目录中建配置文件webpack.config.js(黑窗口中输入webpack后，会自动去找这个文件)
    ③npm i jquery -S
    ④如果想要实现实时打包和构建，需要依赖于 webpack-dev-server 这个工具
    ⑤ 配置webpack-dev-server的步骤：
    运行cnpm i webpack-dev-server webpack -D（本地也需要安装webpack）
    在 package.json的scripts节点下，新增dev脚本，脚本的命令是
    "dev": "webpack-dev-server"

在控制台，运行npm run dev （自动更新）
原理： 通过 webpack-dev-server打包好的bundle.js存放到了内存中，被托管于项目的跟目录中；
因此，如果要访问bundle.js,需要通过`
（根目录中的bundle.js不在物理磁盘中，而在内存中，所以看不见）

  1. 使用html-webpack-plugin插件配置启动页面
    html-webpack-plugin它的作用，就是把index.html页面，也托管到内存中，进一步增加网站在开发过程中，运行的速度；第二个作用：自动把打包好的bundle.js注入到页面中；
    ①运行cnpm i html-webpack-plugin -D 把插件安装到项目的开发依赖中；
    ②在webpack.config.js中，导入html-webpack-plugin， 得到一个构造函数：

// 导入 html-webpack-plugin 它之后，就得到一个构造函数
const HtmlWebpackPlugin = require(‘html-webpack-plugin’)
new 一个实例对象，并在new实例对象的时候，提供一个配置对象：
// 这个插件的作用，就是把 index.html 页面，复制一份，托管到内存中
③ const htmlConfig = new HtmlWebpackPlugin({
template: path.join(__dirname, ‘./src/index.html’), // 指定要复制的页面的路径
filename: ‘index.html’ // 指定要生成的页面的名称（这个生成的页面，被托管到了内存中）,自动生成的页面，被托管到了 项目的根目录中；
})
把 new 出来的实例对象，挂载到 plugins 数组中即可：
plugins: [ // 这个节点，是专门来放置 webpack 相关的插件的
htmlConfig
]
3.实现自动打开浏览器、热更新和配置浏览器的默认端口号
如果要让webpack第一次打包构建好项目以后，自动打开浏览器，可以使用–open参数；
在package.json 中修改
“dev”: “webpack-dev-server –open”

–open chrome
–open firefox
–open iexplore
如果要指定运行的端口号，要使用–port 3000 记得空格
如果要指定运行的IP地址，要使用–host 127.0.0.1
如果要在开发阶段，压缩网络传输时候的文件，要使用–compress
如果要展示编译进度，要使用–progress

4.webpack 默认，只能帮我们打包并处理 .js 文件，但是，无法处理那些 非 .js 文件；
处理css文件
在mian.js中先导入 import ‘./css/index.css’

运行cnpm i style-loader css-loader -D安装合适的loader
找到webpack.config.js配置文件，新增一个module节点，并为module节点提供rules数组匹配规则：
module: { // 这个 module 节点，可以配置好多的第三方loader模块
rules: [ // 这个 rules 是 第三方文件后缀名的匹配规则，在这里，是通过后缀名来匹配规则的 { test: /.css$/, use: [‘style-loader’, ‘css-loader’] }
]
}

  1. 使用webpack打包less文件
    运行cnpm i less-loader less -D安装相关的loader和内置依赖项
    在 module 节点的 rules数组中，添加一个 loader 配置规则：
    { test: /.less$/, use: [‘style-loader’, ‘css-loader’, ‘less-loader’] }
    使用webpack打包scss文件 index.scss（超级样式）
    运行cnpm i sass-loader node-sass -D 安装相关的loader和内置（前置）依赖项
    添加loader匹配规则：
    { test: /.scss$/, use: [‘style-loader’, ‘css-loader’, ‘sass-loader’] }

7.使用webpack处理css中的路径

运行 cnpm i url-loader file-loader -D
添加loader匹配规则：（如果只有一个loader也可以不用写[ ] 直接写名字就行
）
如： { test: /.jpg|png|gif|bmp$/, use: ‘url-loader’ }
可以为url-loader提供limit和name参数，来规定 极限值和 打包之后的图片名称；
?limit=图片的字节（limit值>=图片的字节 ，url值就会转换为base64）
{ test: /.jpg|png|gif|bmp$/, use: ‘url-loader?limit=1&name=[hash:8]-[name].[ext]’ }
其中，[hash:8] 取hash值的前8位； [name]表示源文件名称 ； [ext]表示源文件的后缀名(解决图片重名问题）

8.class关键字，这是 ES6中实现面向对象编程的新语法
// 在 React阶段，我们必须会讲到 class 关键字 ，因为在react中，就是用 class关键字来创建组件的构造函数的
class Person {
static info = { country: ‘China’ }
}
console.log(Person.info)
使用babel处理高级JS语法
webpack 默认只能够打包处理一部分的高级JS语法；
如果要使用babel处理高级的ES语法，需要安装两套包：
运行cnpm i babel-core babel-loader babel-plugin-transform-runtime -D
运行cnpm i babel-preset-env babel-preset-stage-0 -D // preset 是语法的意思

添加loader规则，注意：一定要把 node_modules 添加到 exclude 排除项:
{ test: /.js$/, use: ‘babel-loader’, exclude: /node_modules/ } // exclude 这个选项一定要添加
加 .babelrc 的babel配置文件，里面配置了babel用到到的语法和插件，注意，这个文件中，必须符合JSON规范，因此，不能使用单引号和注释：
{
“presets”: [“env”, “stage-0”],
“plugins”: [“transform-runtime”]
}
